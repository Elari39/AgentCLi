# 实战演练：动手实现命令系统

派大星 6月12日 13:20

## 本章需要做什么？

上一章我们给 MewCode 装上了跨会话记忆系统，它终于能记住你的偏好和项目上下文了。但有些日常操作还是让人哭笑不得——你想清个屏，Agent 认认真真地回复「好的，已帮你清除」，实际上什么也没发生，还白白消耗了 token。

这一章要给 MewCode 装上 Slash Command 内置命令框架。做完之后，所有以 `/` 开头的常用操作都会绕过 Agent Loop 直接在本地执行，省 Token、响应快、行为确定。

具体要新增这些东西：

- **命令框架** ：CommandRegistry（注册/查找/列举/补全）、命令解析器、CommandContext
- **UIController 接口** ：抽象 UI 操作，命令不依赖具体 UI 实现
- **十个内置命令** ：help、compact、clear、plan、do、session、memory、permission、status、review
- **UI 集成** ：命令拦截优先于 Agent 调用、Tab 补全、状态栏

这章 **不做** ：用户自定义命令、动态 prompt 生成、命令权限控制（这些留给后面的 Skill 系统）。

## Vibe Coding 实战

### 生成三份文档

把任务换成本章的内容：

```markdown
# 我的初步想法
- 设计一个集中式注册中心，每条命令带名称 / 别名 / 描述 / 用法 / 类型 / 参数提示 / 处理函数等元数据，注册时检测别名冲突
- 解析器识别 `/` 前缀输入，第一个空格之前是命令名、之后是参数；命令名转小写做到大小写不敏感
- 把命令按执行模式分三类：纯本地、影响 UI 状态、需要把预设提示词送进对话流让 AI 处理
- 抽一层 UI 控制接口（显示系统消息、发送用户消息、切换模式、查 token、刷状态等），让命令实现不依赖具体终端渲染框架
- 在用户回车的事件入口加一个分流器：是命令走本地分发，不是命令才送给 AI
- 支持别名和 Tab 补全：单匹配直接补全、多匹配弹列表，隐藏命令不参与
- 状态栏配合显示当前模式和高频命令提示；未知命令统一引导到帮助入口
- 内置一批高频操作：帮助、上下文压缩、清空对话、模式切换（计划/执行）、会话管理、记忆管理、权限管理、综合状态、代码审查等
```

然后 AI 就会开始问你问题，进行需求澄清。

你根据理论篇学到的内容回答这些问题，一直这样反复循环对齐需求，最后就能生成三份文档了。

### 功能验证过程

来验收一下结果：

启动 MewCode，我们先来看看全局命令概览，输入

> /help

就能看到我们当前所支持的所有命令

![](images/chapter-10-2/img-3.png)

可以看到我们目前拥有的是个命令都展示出来了，大多数我们在前面都验收演示过，挑几个命令我们试下

比如我们的想看看权限管理的样子，就输入

> /permission

![](images/chapter-10-2/img-4.png)

就能把我们当前模式的权限显示出来

又或者我们想看整个项目MewCode的状态，就可以输入

> /status

![](images/chapter-10-2/img-5.png)

我们就能看到我们的权限模式，位于当前token的使用情况、工具注册情况、记忆、工作目录

如果我们想审查代码变更，就可以输入

> /review

![](images/chapter-10-2/img-6.png)

这是提示词命令，执行的时候会直接把我们的这段提示词发给模型，也就是相当于复用prompt，执行起来的效果就如下

![](images/chapter-10-2/img-7.png)

执行命令后就会把我的提示词发送过去，很舒服。

验收没问题，那么本章的主要任务就完成了。

不知道大家有没有隐隐约约地感觉到，命令框架非常稳定还能复用流程和提示词，而Agent链路经常会不稳定比如调错用工具而且无法复用，有没有可能让两者结合呢，下一章，我们的SKill机制会给你答案。
